`safer_ffi` User Guide

Getting started

Cargo.toml

To start using ::safer_ffi, edit your Cargo.toml like so:

[lib]
crate-type = ["staticlib"]  # and/or "cdylib"

[dependencies]
safer-ffi = { version = "...", features = ["proc_macros"] }

[features]
c-headers = ["safer_ffi/headers"]
  • See the dedicated chapter on Cargo.toml for more info.

src/lib.rs

Then, to export a Rust function to FFI, add the #[ffi_export] attribute like so:

use ::safer_ffi::prelude::*;

/// Adds two integers together.
///
/// \remark Wraps on overflow.
#[ffi_export]
fn add (x: i32, y: i32) -> i32
{
    x.wrapping_add(y)
}
  • See the dedicated chapter on #[ffi_export] for more info.

Generating the library (binary file)

Simply run cargo build or cargo build --release to generate the library file at target/{debug,release}/libcrate_name.ext

Generating the C header file

To let safer_ffi generate the corresponding C headers, add also the following to your src/lib.rs:

#[::safer_ffi::cfg_headers]
#[test]
fn generate_headers () -> ::std::io::Result<()>
{
    ::safer_ffi::headers::builder()
        .to_file("filename.h")?
        .generate()
}

and then run the following command:

cargo test --features c-headers -- generate_headers --nocapture

Using it from C

Click to expand

Here is a basic example to showcase FFI calling into our add function:

// main.rs
#include <inttypes.h>
#include <stdio.h>
#include <stdlib.h>

#include "filename.h"

int main (int argc, char const * const argv[])
{
    printf("42 + 27 = %" PRId32 "\n", add(42, 27));
    return EXIT_SUCCESS;
}

compile it with:

cc main.c -o main -L target/debug -l crate_name
./main  # Run it!

Custom Types

Custom types are also supported, as long as they:

  • have a defined C layout;

  • have a #[derive_ReprC] attribute.

Usage with structs

#[derive_ReprC] // <- `::safer_ffi`'s attribute
#[repr(C)]      // <- defined C layout is mandatory!
struct Point {
    x: i32,
    y: i32,
}
  • See the dedicated chapter on structs for more info.

Usage with enums

#[derive_ReprC] // <- `::safer_ffi`'s attribute
#[repr(u8)]     // <- explicit integer `repr` is mandatory!
pub enum Direction {
    Up = 1,
    Down = -1,
}
  • See the dedicated chapter on enums for more info.
